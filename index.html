Software and Tool Development

It often happens that I will encounter a workflow problem or a point of friction. In these situations I have taken to writing my own tools that address those issues. My default language of choice is Rust, but I can pick up whatever is most appropriate for the job. The following are some of the tools I've written for myself.

Monsim

Pok√©mon Battle Simulator

Monsim is a work-in-progress tool that acts as the engine for a Pok√©mon battle simulation. It is meant to be used eventually in a GUI a tool for Pok√©mon fan artists to organise their designs, tweak their in-game stats, and test them out. It is not a game, rather it is a content creation tool.

Monsim has been in development since 2020. The source is available on GitHub.

File-Tags.yazi

Tagging plugin for Yazi file manager

File-Tags is a plugin for the Yazi terminal file manager that allows users to tag files with custom tags and then use that to search for files by tags. It was made for use cases when tree-like file organisation is insufficient and the user wants to group files that naturally belong in different folders.

The source is available at GitHub.

DirDiff

CLI tool for diffing the contents of two directories

DirDiff is a CLI tool that, given two directories, will report the files and folders that are different. It will then generate a bash script that includes commands to resolve the differences by only copying the modified files. The primary use for this is to quickly back up files to an external hard drive, as opposed to copying the entire contents.

The source is available at GitHub.

Clock Widget

Clock Overlay with Low CPU Usage

This unnamed clock widget provides a resizable translucent overlay that includes the current time and date on the screen, made in Godot. I made this to address my own time blindness issues. Having a large time overlay helps keep track of time.

The source is available at GitHub.

Clicky

TUI Typing Tutor

Clicky is a typing practice app that works in the terminal for blazing fast performance. I used to use Keybr.com, but started noticing lag between when I pressed a key and when it would update the GUI. Clicky solves that by not being rendered in the browser.

The source is available at GitHub.
